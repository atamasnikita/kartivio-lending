# Kartivio App Improvement Plan

Дата: 2026-06-28

## Правило процесса

По UI идем последовательно. Перед кодом каждого шага сначала фиксируем мини-спеку:

1. Что меняем.
2. Почему это должно улучшить продукт.
3. Какие экраны и файлы затрагиваем.
4. Как будет выглядеть на мобильном, в миниаппе и на десктопе.
5. Как проверяем результат.

Кодим только после отдельного подтверждения.

## Общие принципы

- Сохраняем текущий темный, фотоцентричный стиль: больше воздуха, меньше лишних рамок и технического текста.
- Главные пользовательские действия должны быть видны без объяснений: создать, повторить, оплатить, открыть результат.
- Не перегружаем экран карточками внутри карточек.
- Для мобильного интерфейса держим стабильные размеры блоков, чтобы текст, загрузки и анимации не прыгали.
- Анимации оставляем только там, где они объясняют состояние или делают ожидание мягче.
- Проверяем 375px, 768px, 1024px и десктоп; отдельно смотрим миниаппу Telegram.

## Очередь работ

### 1. Быстрый технический шаг без изменения UI

Статус: выполнено 2026-06-28.

Цель: ускорить первый вход, открытие миниаппы и переключение ленты.

Что сделать:
- включить сжатие статики на сайте;
- облегчить загрузку ленты, чтобы клиент не получал тяжелые данные, которые не нужны для первого экрана;
- проверить, что это не ломает оплату, миниаппу и web-версию.

Почему первым:
- это даст прирост скорости без визуального риска;
- после этого будет честнее оценивать UI, потому что часть раздражения может быть от медленной загрузки.

Результат:
- статика сайта отдается с gzip;
- `app.js` по сети стал около 52 KB вместо 255 KB;
- `/v1/templates?include_prompt=false` отдает каталог без промптов;
- продовый ответ шаблонов стал около 75 KB вместо 531 KB;
- промпт конкретного шаблона догружается по `/v1/templates/{template_id}`.

### 2. История генераций

Цель: сделать историю не техническим списком, а понятной галереей результатов.

Идея:
- карточка результата должна быть image-first;
- тап по картинке открывает результат;
- основные действия: повторить, открыть, скачать там, где это корректно;
- в миниаппе не показывать действия, которые плохо работают в Telegram;
- статусы генерации показывать компактно и без визуального шума.

### 3. Экран генерации

Цель: сделать создание изображения проще и понятнее.

Идея:
- собрать экран вокруг понятной последовательности: сцена/шаблон, свои фото, промпт, параметры, генерация;
- настройки модели и формата свернуть в компактную строку-сводку;
- кнопку генерации сделать стабильной и заметной, особенно на мобильном;
- блок промпта по референсу оставить премиальным, но не давать ему визуально спорить с основным действием.

### 4. Лента и категории

Цель: сохранить красоту ленты, но подготовить ее к росту числа категорий.

Проблема:
- горизонтальная строка чипсов будет становиться все длиннее;
- пользователю будет сложнее понять, где нужная категория;
- новые категории начнут конкурировать с важными: Новое, Все темы, Избранное.

Идеи для проработки:
- оставить быстрые чипсы: Новое, Все темы, Избранное и 3-5 самых важных;
- остальные категории увести в компактный каталог/меню;
- сгруппировать категории по смыслу: люди, стиль, семья, город, праздники, полезности;
- добавить визуальный акцент для новых шаблонов без перегруза;
- проверить, нужна ли отдельная строка "подборки" вместо бесконечных чипсов.

### 5. Модалка шаблона

Цель: сделать выбор шаблона более продающим.

Идея:
- крупнее показывать изображение;
- основной CTA: создать с этим шаблоном;
- промпт спрятать под раскрытие "Посмотреть промпт";
- похожие шаблоны или категория могут быть вторым этапом, если не перегрузит.

### 6. Десктопная адаптация

Цель: чтобы web-версия выглядела не как растянутая мобильная миниаппа.

Идея:
- ограничить рабочую ширину;
- для ленты и истории использовать более плотные сетки;
- для генерации использовать двухколоночную компоновку: ввод слева, предпросмотр/параметры справа.

### 7. Профиль и кредиты

Цель: сделать профиль утилитарным и понятным.

Идея:
- сверху баланс и быстрый CTA пополнения;
- ниже аккаунт, способ входа, уведомления Telegram;
- убрать ощущение служебной страницы.

## Отложенные идеи

- Полноценный UI для самостоятельной загрузки шаблонов уже есть в базе как направление, но пока не смешиваем с текущим UI-проходом.
- Пейволы под разные сценарии можно собрать отдельно после стабилизации основного приложения.
- Рассылки и офферы не трогаем в этом цикле, если нет новой проблемы.
